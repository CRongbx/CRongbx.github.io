概述

​ CS架构的网络通信过程。这里有几个概念：

  • 守护程序（daemon）
    Web服务器长时间运行的一个程序，用来响应来自网络的客户端发来的请求。
  • 异步回调（asynchronous callback）通信
    由服务器向客户端发起的通信请求。(大多数情况下都是客户先发起请求，这样设计协议和编写程序都比较简单)。

​ 本书聚焦点：传输层和网络层协议。重点介绍TCP/IP协议族，也会涉及UDP协议。

​ client和server通过TCP/IP在同一个局域网通信的整个过程如下图。这里主要强调网络中“层”的概念，即应用程序通过TCP向服务器通信，TCP又转而使用下层IP通信，IP在通过某种形式的数据链路层通信。具体TCP四层协议的知识可以参考《计算机网络》。

下面给出Linux下常见的文件后缀意义^[3]^：

.o文件（目标文件）：经过：预编译（预处理）、编译和汇编，形成的二进制文件。

.a文件：静态链接库文件。将若干.o文件静态打包（链接？）后形成的二进制文件。

.so文件：动态链接库文件。遵循共享内存机制。

.lo文件：使用libtool编译出的目标文件，其实就是在.o文件中添加了一些信息。

.la文件：使用libtool编译链接出的库文件，其实就是个文本文件，记录同名动态库和静态库的相关信息

同时书中的源代码也出现了.lc和.lh后缀的文件，如下图。这类后缀的文件(.lx)是将原本.x文本文件增加了行号、路径等信息而成的文本文件，为了方便编译，但是本质上和source code没有差别。

C中define和const的区别

define：定义的对象为常数，常数在编译时展开

const：定义的对象为常量， 常量在运行时初始化后确定^[4][5]^

define const
定义对象 常数 常量
合适确定 预编译时展开 运行初始化时
是否有类型检查 无 有
空间占用 占用代码空间 占用运行时数据段空间
调试 不可调（预编译时替换） 可以
可否重新定义 可以（#undef取消） 不能

创建TCP socket

函数原型：

  • : 设置网络通信的域，socket()通过它选择通信协议的族。常用的domain有：

    | 值 | 含义 |
    | :——————— | :———————————————————————- |
    | EACCES | 没有权限建立制定的domain的type的socket |
    | EAFNOSUPPORT | 不支持所给的地址类型 |
    | EINVAL | 不支持此协议或者协议不可用 |
    | EMFILE | 进程文件表溢出 |

为什么要循环read

需要注意TCP是个没有记录边界的字节流协议，比如服务器的应答可能是：

​ \r是ASCII中的回车符（回到行首）；\n是ASCII中的换行符。

​ 使用TCP传输上面的字符时服务器可能有多种返回方式：

  1. 1个包含所有内容的TCP segment（分段）
  2. 若干包含a字节（a<消息总长度）的TCP分段

所以如果数据量很大，我们不能确保TCP只用一个分段就传送所有数据，即可能将data拆分为多段传送，所以我们总是需要把read()写在某个循环中，当它读完（0）或出现错误（<0）时终止循环。

ipv4的客户端程序’daytimetcpcli’对应的服务端程序’daytimetcpsrv’. ipv4中本地回环地址’127.0.0.1’

ipv6的客户端程序’’daytimetcpcliv6’对应的服务端程序’daytimetcpsrvv6’. ipv6中本地回环地址’0:0:0:0:0:0:0:1’

错误处理：包裹函数（Wrapper function）

Wrapper function作用

​ 简化每次调用函数、判断返回值、错误处理的一系列操作。每个Wrapper function可以一次性完成这3个过程。这样在编写代码时，可以使程序更加简洁明了。

注意几点:

  • accept()服务器在未有连接时属于睡眠状态，等待某个client连接的到达并接受，为每个连接到本服务器的客户返回一个新描述符。使用TCP三次握手创建连接手，返回accept的新描述符（connected descriptor） 。服务器使用这个新描述符与建立连接的客户通信。
  • ctime()将time_t类型时间转换为直观可读的时间格式。
  • server的创建的socketIIP地址设置为INADDR_ANY表示server可以在任意网络接口上接受client连接。

本程序的局限性

  • 只适用于IPV4
  • 服务器一次只能处理一个客户。【多线程】
