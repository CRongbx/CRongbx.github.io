Makefile简介

​ make命令执行时，需要一个makefile文件，以告诉make命令需要怎么样的去编译和链接程序。

​ 为什么要编写Makefile呢？Windows或macOS环境下，有VS2015, Eclipse, XCode等优秀的IDE作为开发工具，只需要点击下编译运行的按钮就可以运行程序了。但是Unix环境下的开发工作，没有IDE，怎么样才能编译运行我们写的代码呢？编写Makefile“自动编译”。

​ 比较权威的文档是GUN make手册。

Makefile的规则

​ Makefile的由一系列规则（rules）组成。每条规则的形式如下：

  • target: 指明make命令要构建的对象
  • prerequisites: 指明生成target需要依赖的库和对象，以及target是否需要重建的判断标准
  • command: 指明该target要执行的shell命令

makefile运行的核心规则是：
==prerequisites中如果有一个以上的文件比target文件要新的话，command所定义的命令就会被执行。==

\目标

​ 一个target就构成了一条规则。目标通常是文件名或伪目标，指明make命令要构建的对象是什么。如果目标是多个文件名，则每个文件用空格分隔开。

Makefile的语法

关闭回声

make命令默认执行时会打印每条命令（echoing回声），若不想打印，在命令前加上@即可。

变量和赋值

Makefile使用= 自定义变量并赋值。

使用变量时要将变量放在$()内.

变量之间也可以互相赋值。

Makefile提供了4种赋值运算符：

  • =在执行时拓展，允许递归扩展
  • :=在定义时扩展
  • ?=只有在该变量为空时才设置值
  • +=将值追加到变量尾端

make如何工作

在默认的方式下，也就是我们只输入 make 命令。那么，

  1. make会在当前目录下找名字叫“Makefile”或“makefile”的文件。
  2. 如果找到，它会找文件中的第一个目标文件（target），在上面的例子中，他会找到“edit”这个 文件，并把这个文件作为最终的目标文件。
  3. 如果edit文件不存在，或是edit所依赖的后面的 .o 文件的文件修改时间要比 edit 这个 文件新，那么，他就会执行后面所定义的命令来生成 edit 这个文件。
  4. 如果 edit 所依赖的 .o 文件也不存在，那么make会在当前文件中找目标为 .o 文件 的依赖性，如果找到则再根据那一个规则生成 .o 文件。（这有点像一个堆栈的过程）
  5. 当然，你的C文件和H文件是存在的啦，于是make会生成 .o 文件，然后再用 .o 文件生 成make的终极任务，也就是执行文件 edit 了。

gcc和cc的区别

​ 仔细阅读上面的Makefile，我们会发现编译的命令使用的是cc而不是gcc。这两个有什么区别？

cc是Unix下使用的编译命令；

gcc是Linux下使用的编译命令。

​ 由于许多Makefile文件是在Unix下编写的，即使用cc编译。但现在许多人喜欢用Linux来编译，所以有人想到在Linux上用连接的方法把cc连接到gcc命令上，即运行cc就是在运行gcc。
